# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

- **User Request**: 디지털 테이블오더 서비스 구축 (고객 주문 + 관리자 모니터링)
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (프론트엔드 + 백엔드 + 데이터베이스)
- **Complexity**: Complex (다중 사용자 유형, 실시간 기능, 세션 관리)
- **Depth**: Standard

---

## 1. 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| 백엔드 | Node.js + Express | SSE 기반 실시간 통신 |
| 프론트엔드 | React + Vite | 단일 앱, 경로 분리 (/customer, /admin) |
| 데이터베이스 | SQLite | better-sqlite3 또는 sqlite3 패키지 |
| 인증 | JWT + bcrypt | 관리자 16시간 세션, 테이블 자동 로그인 |
| 실시간 통신 | Server-Sent Events (SSE) | 주문 모니터링용 |
| 배포 | 로컬 개발 우선 | 추후 배포 환경 결정 |

---

## 2. 비즈니스 컨텍스트

- **매장 구조**: 단일 매장 전용 (데이터 모델은 매장 식별자 포함하여 확장 가능)
- **접속 규모**: 소규모 (테이블 10개 이하)
- **관리자 계정**: DB seed로 사전 설정 (회원가입 기능 없음)
- **이미지 관리**: 외부 URL 직접 입력 (업로드 기능 없음)
- **보안 확장**: 미적용 (기본 보안만 - JWT, bcrypt)

---

## 3. 기능 요구사항

### 3.1 고객용 기능 (Customer)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정: 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 로그인 정보 로컬 저장 (localStorage)
- 이후 자동 로그인 (저장된 정보로 인증)

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(메인) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 저장 (localStorage, 새로고침 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 주문 성공: 주문 번호 표시 (5초) -> 장바구니 비우기 -> 메뉴 화면 리다이렉트
- 주문 실패: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장ID, 테이블ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시 (이전 세션 제외)
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 페이지네이션 또는 무한 스크롤

### 3.2 관리자용 기능 (Admin)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱, 로그인 시도 제한
- 관리자 계정은 DB seed로 사전 설정

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트 (2초 이내)
- 그리드/대시보드 레이아웃: 테이블별 카드
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조 (색상, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호/비밀번호 설정, 16시간 세션 생성
- **주문 삭제**: 확인 팝업 -> 즉시 삭제 -> 총 주문액 재계산
- **테이블 세션 처리**: 이용 완료 시 주문 내역을 과거 이력으로 이동, 현재 목록/총액 리셋
- **과거 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (조회/등록/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

## 4. 비기능 요구사항

### NFR-01: 성능
- SSE 기반 실시간 주문 알림 2초 이내 전달
- 소규모(테이블 10개 이하) 환경에서 원활한 동작

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 터치 타겟)
- 명확한 시각적 계층 구조
- 카드 형태 메뉴 레이아웃

### NFR-03: 보안
- JWT 토큰 기반 인증
- bcrypt 비밀번호 해싱
- 로그인 시도 제한

### NFR-04: 데이터 관리
- 장바구니: 클라이언트 localStorage
- 주문/메뉴/세션: SQLite 데이터베이스
- 세션 ID로 주문 그룹화
- 테이블 이용 완료 시 OrderHistory로 이동

---

## 5. 제외 사항 (Constraints)

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 업로드/리사이징/최적화
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주문 전달, 재고 관리)
- 고급 기능 (분석, 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달, POS, SNS, 지도, 번역)

---

## 6. UI 구성

- **단일 React 앱**, 경로 기반 분리
  - `/customer/*` - 고객용 인터페이스 (메뉴, 장바구니, 주문, 주문내역)
  - `/admin/*` - 관리자용 인터페이스 (로그인, 대시보드, 메뉴관리, 테이블관리)
