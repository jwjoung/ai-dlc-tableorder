# 테이블오더 서비스 - 사용자 스토리

## 스토리 형식
- **사용자 여정 기반** 분해
- **Checklist** 수용 기준
- **사용자 여정 순서** 우선순위 (핵심 플로우 먼저)

---

# 고객 여정 (Customer Journey)

## US-C01: 테이블 자동 로그인

**As a** 고객,
**I want to** 테이블 태블릿에서 별도 조작 없이 자동으로 로그인되기를,
**So that** 즉시 메뉴를 확인하고 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 관리자가 초기 설정한 매장 식별자, 테이블 번호, 비밀번호가 localStorage에 저장된다
- [ ] 저장된 정보가 있으면 페이지 로드 시 자동으로 로그인 API를 호출한다
- [ ] 로그인 성공 시 메뉴 화면으로 이동한다
- [ ] 로그인 실패 시 초기 설정 화면을 표시한다
- [ ] 저장된 정보가 없으면 초기 설정 화면(매장ID, 테이블번호, 비밀번호 입력)을 표시한다

**Persona**: 고객 (Customer)

---

## US-C02: 메뉴 조회 및 탐색

**As a** 고객,
**I want to** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를,
**So that** 원하는 메뉴를 쉽게 찾아 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 화면이 기본(메인) 화면으로 표시된다
- [ ] 카테고리 탭/버튼으로 메뉴가 분류되어 표시된다
- [ ] 각 메뉴 카드에 메뉴명, 가격, 이미지가 표시된다
- [ ] 메뉴 카드 클릭 시 상세 정보(설명 포함)가 표시된다
- [ ] 카테고리 간 빠른 이동이 가능하다
- [ ] 터치 타겟이 최소 44x44px 이상이다

**Persona**: 고객 (Customer)

---

## US-C03: 장바구니에 메뉴 추가

**As a** 고객,
**I want to** 원하는 메뉴를 장바구니에 담기를,
**So that** 여러 메뉴를 한번에 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 상세 화면에서 장바구니 추가 버튼이 있다
- [ ] 장바구니에 담긴 메뉴의 수량을 증가/감소할 수 있다
- [ ] 장바구니에서 특정 메뉴를 삭제할 수 있다
- [ ] 장바구니 전체 비우기가 가능하다
- [ ] 총 금액이 실시간으로 계산되어 표시된다
- [ ] 장바구니 데이터가 localStorage에 저장되어 새로고침 후에도 유지된다

**Persona**: 고객 (Customer)

---

## US-C04: 주문 생성 및 확인

**As a** 고객,
**I want to** 장바구니의 메뉴를 주문으로 확정하기를,
**So that** 매장에서 음식을 받을 수 있다.

**Acceptance Criteria:**
- [ ] 장바구니 화면에서 주문 내역(메뉴명, 수량, 단가, 총액)을 최종 확인할 수 있다
- [ ] 주문 확정 버튼을 누르면 서버에 주문이 전송된다
- [ ] 주문 성공 시 주문 번호가 5초간 표시된다
- [ ] 주문 성공 후 장바구니가 자동으로 비워진다
- [ ] 5초 후 메뉴 화면으로 자동 리다이렉트된다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다
- [ ] 주문 정보에 매장ID, 테이블ID, 세션ID가 포함된다

**Persona**: 고객 (Customer)

---

## US-C05: 주문 내역 조회

**As a** 고객,
**I want to** 현재 테이블 세션의 주문 내역을 확인하기를,
**So that** 내가 주문한 항목과 상태를 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 현재 테이블 세션의 주문만 표시된다 (이전 세션 제외)
- [ ] 주문이 시간 순으로 정렬된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴/수량, 금액이 표시된다
- [ ] 주문 상태(대기중/준비중/완료)가 표시된다
- [ ] 주문 목록이 페이지네이션 또는 무한 스크롤로 제공된다

**Persona**: 고객 (Customer)

# 관리자 여정 (Admin Journey)

## US-A01: 매장 관리자 로그인

**As a** 관리자,
**I want to** 매장 식별자와 계정 정보로 로그인하기를,
**So that** 매장 관리 시스템에 접근할 수 있다.

**Acceptance Criteria:**
- [ ] 매장 식별자, 사용자명, 비밀번호 입력 폼이 제공된다
- [ ] 로그인 성공 시 JWT 토큰이 발급되고 저장된다
- [ ] 세션이 16시간 동안 유지된다
- [ ] 브라우저 새로고침 시에도 세션이 유지된다
- [ ] 16시간 후 자동 로그아웃된다
- [ ] 잘못된 자격증명 시 에러 메시지가 표시된다
- [ ] 로그인 시도 횟수가 제한된다

**Persona**: 관리자 (Admin)

---

## US-A02: 실시간 주문 모니터링

**As a** 관리자,
**I want to** 들어오는 주문을 실시간 대시보드에서 확인하기를,
**So that** 빠르게 주문을 처리할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블별 카드가 그리드 레이아웃으로 표시된다
- [ ] 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
- [ ] SSE를 통해 신규 주문이 2초 이내에 화면에 반영된다
- [ ] 신규 주문 시 시각적 강조(색상 변경, 애니메이션)가 적용된다
- [ ] 테이블 카드 클릭 시 해당 테이블의 전체 주문 목록이 상세 표시된다
- [ ] 테이블별 필터링이 가능하다

**Persona**: 관리자 (Admin)

---

## US-A03: 주문 상태 변경

**As a** 관리자,
**I want to** 주문 상태를 변경하기를,
**So that** 주문 처리 진행 상황을 추적하고 고객에게 알릴 수 있다.

**Acceptance Criteria:**
- [ ] 각 주문에 상태 변경 버튼이 제공된다
- [ ] 대기중 → 준비중 → 완료 순서로 상태 변경이 가능하다
- [ ] 상태 변경 시 화면에 즉시 반영된다
- [ ] 상태별 시각적 구분이 명확하다

**Persona**: 관리자 (Admin)

---

## US-A04: 테이블 초기 설정

**As a** 관리자,
**I want to** 테이블 번호와 비밀번호를 설정하기를,
**So that** 고객 태블릿이 해당 테이블로 자동 로그인할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블 번호와 비밀번호를 입력/설정할 수 있다
- [ ] 설정 완료 시 16시간 테이블 세션이 생성된다
- [ ] 설정 정보가 저장되어 태블릿 자동 로그인이 활성화된다
- [ ] 성공/실패 피드백이 표시된다

**Persona**: 관리자 (Admin)

---

## US-A05: 주문 삭제

**As a** 관리자,
**I want to** 특정 주문을 삭제하기를,
**So that** 잘못된 주문을 직권으로 수정할 수 있다.

**Acceptance Criteria:**
- [ ] 각 주문에 삭제 버튼이 제공된다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 확인 시 주문이 즉시 삭제된다
- [ ] 삭제 후 해당 테이블의 총 주문액이 재계산된다
- [ ] 성공/실패 피드백이 표시된다

**Persona**: 관리자 (Admin)

---

## US-A06: 테이블 이용 완료 처리

**As a** 관리자,
**I want to** 테이블 세션을 종료(이용 완료)하기를,
**So that** 새 고객이 깨끗한 상태에서 주문을 시작할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블 카드에 이용 완료 버튼이 제공된다
- [ ] 이용 완료 전 확인 팝업이 표시된다
- [ ] 확인 시 해당 세션의 주문 내역이 과거 이력(OrderHistory)으로 이동된다
- [ ] 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다
- [ ] 새 고객은 이전 주문 내역 없이 시작할 수 있다
- [ ] 성공/실패 피드백이 표시된다

**Persona**: 관리자 (Admin)

---

## US-A07: 과거 주문 내역 조회

**As a** 관리자,
**I want to** 테이블별 과거 주문 내역을 조회하기를,
**So that** 이전 고객들의 주문 이력을 확인할 수 있다.

**Acceptance Criteria:**
- [ ] 과거 내역 버튼 클릭 시 과거 주문 목록이 표시된다
- [ ] 시간 역순으로 정렬된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
- [ ] 날짜 필터링이 가능하다
- [ ] 닫기 버튼으로 대시보드에 복귀할 수 있다

**Persona**: 관리자 (Admin)

---

## US-A08: 메뉴 등록

**As a** 관리자,
**I want to** 새로운 메뉴를 등록하기를,
**So that** 고객이 새 메뉴를 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴명, 가격, 설명, 카테고리, 이미지 URL을 입력할 수 있다
- [ ] 필수 필드(메뉴명, 가격, 카테고리) 검증이 수행된다
- [ ] 가격 범위 검증이 수행된다 (0 이상)
- [ ] 등록 성공 시 메뉴 목록에 즉시 반영된다
- [ ] 성공/실패 피드백이 표시된다

**Persona**: 관리자 (Admin)

---

## US-A09: 메뉴 수정 및 삭제

**As a** 관리자,
**I want to** 기존 메뉴를 수정하거나 삭제하기를,
**So that** 메뉴 정보를 최신 상태로 유지할 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 목록에서 수정/삭제 버튼이 제공된다
- [ ] 수정 시 기존 정보가 폼에 미리 채워진다
- [ ] 수정 사항 저장 시 필수 필드/가격 검증이 수행된다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 변경 사항이 메뉴 목록에 즉시 반영된다

**Persona**: 관리자 (Admin)

---

## US-A10: 메뉴 노출 순서 조정

**As a** 관리자,
**I want to** 메뉴의 노출 순서를 조정하기를,
**So that** 고객에게 원하는 순서로 메뉴를 보여줄 수 있다.

**Acceptance Criteria:**
- [ ] 카테고리별 메뉴 목록에서 순서 조정이 가능하다
- [ ] 순서 변경이 고객 화면에 즉시 반영된다
- [ ] 성공/실패 피드백이 표시된다

**Persona**: 관리자 (Admin)
